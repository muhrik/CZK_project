My bachelor's thesis Oris 2 / fdPRISM stochastic model benchmark:

The practical output of my thesis consisted of searching and evaluating academic publications for DSPNs (Deterministic and Stochastic Petri Nets)
which I then implemented as fdPRISM fdCTMCs (fixed delay Continuous Time Markov Chains). Some of them I slightly modified (if suitable).
I also recreated the DSPNs in Oris 2 tool and wrote a readme file about each model, which contains the source and explanation of the model.
Afterwards I created this webpage and the spreadsheet, so that the models could be conveniently browsed by their properties.

The primary intention is:
  • to provide convenient and well-documented collection of stochastic models implemented in both fdPRISM and Oris 2

    • The primary usage is:
  • a benchmark for tools used for probabilistic modeling, verification and analysis of real-time systems (namely fdPRISM and Oris 2)

    • Secondary use cases might be:
  • as a compilation of DSPN implementations in Oris 2
  • as a compilation of academic publications that had DSPNs in them
  • to demonstrate fdPRISM fdCTMC is indeed equivalent formalism to DSPN
  • to showcase fdPRISM fdCTMC implementation examples

    •
    Explanation for some of the columns in the spreadsheet:
    • Num_Modifiable_Parameters - amount of modifiable parameters. They mostly affect the state space. Sortable.
    • Modifiable_Parameters - modifiable parameters themselves. They contain the name and default value of the parameter.
    • Num_Default_Tangible_States - amount of reachable states given for DEFAULT parameters. State space can be changed through the parameters.
    • Num_Det-Trans - amount of deterministic (fixed delay) transitions in the model.
    • Num_Max_Concurrent_NonExp-Trans - maximum amount of transitions that are not exponential (therefore, not memoryless) that may be activated simultaneously at any given time
    • Is_Regenerative - whether the model is regenerative. Regenerative models have some memoryless configurations they are guaranteed to reach repeatedly.
    • fdPRISM_mean_payoff-ready - whether the fdPRISM fdCTMC model is suitable for Mean Payoff analysis AND has costs/rewards implemented.
    • fdPRISM_fd_synthesis-ready - whether the fdPRISM fdCTMC model is suitable for fixed-delay synthesis AND has costs/rewards implemented.
